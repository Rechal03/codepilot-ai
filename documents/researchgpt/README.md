# ResearchGPT — AI Research Assistant

ResearchGPT is a Retrieval-Augmented Generation (RAG) application that lets you upload documents and ask questions about them in plain English. It retrieves the most relevant sections of your document and uses a locally-hosted LLM to generate accurate, source-grounded answers — no hallucinated information, no cloud API costs.
## Demo

## Features

- Upload PDF or DOCX documents
- Automatic text extraction and chunking
- Semantic search using vector embeddings (not just keyword matching)
- Answers generated by a local LLM (Llama 3 via Ollama) — fully private, runs on your own machine
- Answers are grounded in the source document, with retrieved source chunks returned alongside each answer
- REST API built with FastAPI

## Tech Stack

- **Backend:** FastAPI, Uvicorn
- **Document Processing:** pypdf, python-docx, BeautifulSoup
- **Text Chunking:** LangChain (RecursiveCharacterTextSplitter)
- **Embeddings:** Hugging Face Sentence Transformers (`all-MiniLM-L6-v2`)
- **Vector Search:** FAISS
- **LLM:** Llama 3, served locally via Ollama
- **Language:** Python

## How It Works

1. **Upload** — A PDF or DOCX file is uploaded through the `/upload` endpoint. The text is extracted, split into overlapping chunks (~500 characters each), converted into vector embeddings, and stored in a FAISS index.
2. **Query** — A question is sent to the `/query` endpoint. The question is embedded into a vector, and FAISS retrieves the most semantically similar chunks from the document.
3. **Answer Generation** — The retrieved chunks are passed to a local Llama 3 model along with the question, with instructions to answer only using the provided context. This prevents hallucination and keeps answers grounded in the actual document.

## Setup

### Prerequisites
- Python 3.10+
- [Ollama](https://ollama.com) installed, with the `llama3` model pulled (`ollama pull llama3`)

### Installation

```bash
git clone https://github.com/Rechal03/researchgpt.git
cd researchgpt
python -m venv .venv
.venv\Scripts\Activate   # Windows
pip install -r requirements.txt
```

### Running the app

```bash
cd app
uvicorn main:app --reload
```

Then open `http://127.0.0.1:8000/docs` to test the API endpoints interactively.
### Running with Docker

Alternatively, run the entire application in an isolated container:

```bash
docker build -t researchgpt .
docker run -p 8000:8000 researchgpt
```

Then open `http://127.0.0.1:8000/docs` to test the API.

**Note:** This container runs the FastAPI backend only. Ollama must be running on your host machine, as the container connects to it for LLM inference.

## API Endpoints

| Endpoint | Method | Description |
|---|---|---|
| `/upload` | POST | Upload a PDF or DOCX file for processing |
| `/query` | POST | Ask a question about the uploaded document |

## Example

**Request:** `POST /query`
question: "What AWS experience does this person have?"

**Response:**
```json
{
  "question": "What AWS experience does this person have?",
  "answer": "According to the context, this person has experience with EC2, EKS, ECS, IAM, KMS, and Secrets Manager...",
  "sources": ["...retrieved chunk 1...", "...retrieved chunk 2..."]
}
```
